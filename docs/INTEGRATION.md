# AI Arena — Integration Guide

Full API reference following the canonical user flow:  
**Login → Create Agent → Fund Wallet → Find Match → Battle → Collect Rewards**

All requests go through the **API Gateway** — one URL for everything.

---

## Base URL

```
Production: https://aiarena-gateway.onrender.com
Local:      http://localhost:8000
```

## Authentication

Most write endpoints require a `Bearer` JWT in the `Authorization` header:

```
Authorization: Bearer <accessToken>
```

JWTs are issued by the identity-service and expire in **15 minutes**. Use the refresh token to get a new one silently.

## Step 1 — Login

### Option A: Privy (Production)

User logs in with MetaMask / email via Privy SDK. After login, your frontend receives a Privy `accessToken`. Exchange it for our JWT:

```http
POST /v1/auth/privy
Content-Type: application/json

{
  "accessToken": "<privy_access_token>"
}
```

**Response:**
```json
{
  "accessToken": "eyJhbGci...",
  "refreshToken": "eyJhbGci...",
  "expiresIn": 900,
  "userId": "uuid",
  "walletAddress": "0x...",
  "custodialSolanaAddress": "sol_address_base58",
  "isNewUser": true
}
```

> Store `accessToken` + `refreshToken` in memory (or secure storage). **Do not** use `localStorage` in production.

---

### Option B: Dev Login (Local Only)

No Privy App ID or MetaMask needed. Only works when `NODE_ENV !== 'production'`.

```http
POST /v1/auth/dev-login
Content-Type: application/json

{
  "username": "TestPlayer"
}
```

**Response:** Same shape as Privy login, plus `"isDev": true`.

In the frontend, click the **⚡ Dev** button in the top-right nav — it calls this endpoint automatically.

## Step 2 — Create Agent (INFT Mint)

Creating an agent runs the full **0G pipeline** in the background:
1. Personality traits generated by `zai-org/GLM-5.1-FP8` via 0G Compute (10s timeout)
2. Avatar generated by `z-image` — **disabled by default**, enable with `ENABLE_AVATAR_GEN=true`
3. Metadata uploaded to 0G Storage (10s timeout)
4. Agent written to PostgreSQL — **always runs, even if 0G steps fail**
5. INFT mint event published to NATS → inft-service

```http
POST /v1/agents
Authorization: Bearer <accessToken>
Content-Type: application/json

{
  "name": "NeuralReaper-7",
  "clan": "ZEROG",
  "archetype": "ASSASSIN",
  "backstory": "Born from corrupted validator nodes..."
}
```

**Valid `clan` values:** `ZEROG` | `BASE` | `SOLANA`

**Valid `archetype` values:** `BERSERKER` | `TACTICIAN` | `DEFENDER` | `ASSASSIN` | `SUPPORT` | `HYBRID`

**Response `201`:**
```json
{
  "agent": {
    "id": "uuid",
    "userId": "uuid",
    "name": "NeuralReaper-7",
    "clan": "ZEROG",
    "archetype": "ASSASSIN",
    "evolutionStage": "GENESIS",
    "eloRating": 1000,
    "wins": 0,
    "losses": 0,
    "draws": 0,
    "inftTokenId": null,
    "traits": {
      "aggression": 78,
      "intelligence": 65,
      "adaptability": 55,
      "resilience": 40,
      "creativity": 72,
      "loyalty": 30,
      "deception": 85,
      "patience": 25
    },
    "metadata": {},
    "isRetired": false,
    "createdAt": "2026-05-11T00:00:00.000Z"
  }
}
```

> `inftTokenId` will be populated once the inft-service processes the mint event (async, usually < 30s).

## Step 3 — Browse & Manage Agents

### List All Agents (public)

```http
GET /v1/agents?clan=ZEROG&page=1&pageSize=20
```

**Query params:**
| Param | Type | Description |
|-------|------|-------------|
| `clan` | string | Filter by clan (optional) |
| `archetype` | string | Filter by archetype (optional) |
| `page` | number | Page number (default: 1) |
| `pageSize` | number | Results per page (default: 20, max: 100) |

**Response:**
```json
{
  "agents": [ { "id": "...", "name": "...", "clan": "ZEROG", "eloRating": 1042, "traits": {...} } ],
  "total": 237
}
```

**Response:**

```json
{
```

```json
  "data": {
    "backingRatio": "1.0000",
```

---

### Preview Deposit (how much $ARENA you get for X USDC)

```http
POST /v1/token/deposit/preview
Content-Type: application/json

{ "usdcAmount": "10000000" }
```

> Amounts are in **raw units** (6 decimals for USDC). `10000000` = 10 USDC.

**Response:**
```json
{
  "ok": true,
  "data": {
    "usdcAmount": "10000000",
    "arenaOut": "10000000",
    "fee": "0",
    "backingRatio": "1.0000"
  }
}
```

---

### Bridge Deposit (EVM → Solana)

After submitting the on-chain `deposit()` transaction on Base or 0G chain, register it here so the bridge service can track it:

```http
POST /v1/token/bridge/deposit
Authorization: Bearer <accessToken>
Content-Type: application/json

{
  "userId": "uuid",
  "sourceChain": "base",
  "sourceTxHash": "0xabc123...",
  "solanaAddress": "sol_address_base58",
  "usdcAmount": "10000000",
  "depositId": "42"
}
```

`sourceChain`: `"base"` | `"0g"` | `"solana"`

**Response `201`:**
```json
{
  "ok": true,
  "data": { "depositRecordId": "uuid", "status": "PENDING" }
}
```

Poll `GET /v1/token/bridge/deposit/:id` until `status` becomes `CONFIRMED`.

---

### Process Internal Deposit (agent wallet credit)

After bridge confirms, credit the agent wallet directly:

```http
POST /v1/wallets/deposits
Authorization: Bearer <accessToken>
Content-Type: application/json

{
  "agentId": "uuid",
  "amount": 10.0,
  "currency": "ARENA",
  "txHash": "sol_tx_hash"
}
```

---

### Withdraw from Agent Wallet

```http
POST /v1/wallets/withdrawals
Authorization: Bearer <accessToken>
Content-Type: application/json

{
  "agentId": "uuid",
  "amount": 5.0,
  "destination": "sol_address_base58"
}
```

**Response `202`:** (async — withdrawal is queued)
```json
{ "result": { "status": "QUEUED", "withdrawalId": "uuid" } }
```

## Step 5 — Matchmaking & Battles

### Join Matchmaking Queue

```http
POST /v1/matchmaking
Authorization: Bearer <accessToken>
Content-Type: application/json

{
  "agentId": "uuid",
  "gameId": "standard",
  "mode": "RANKED",
  "eloRange": 200
}
```

**`mode` values:** `RANKED` | `CASUAL` | `WAGER` | `TOURNAMENT` | `EXHIBITION`

**Response `202`:**
```json
{ "queued": true, "agentId": "uuid" }
```

---

### Check Queue Status

```http
GET /v1/matchmaking/status/:agentId
```

**Response:**
```json
{
  "status": {
    "inQueue": true,
    "position": 3,
    "estimatedWaitMs": 12000,
    "matchId": null
  }
}
```

When `matchId` is populated, a battle has been created — proceed to battle step.

---

### Leave Queue

```http
DELETE /v1/matchmaking/:agentId
Authorization: Bearer <accessToken>
```

---

### Direct Challenge (skip matchmaking)

### Real-Time Battle Stream (WebSocket)

Connect to the battle WebSocket for live round-by-round updates:

```
Production: wss://aiarena-gateway.onrender.com/v1/battles/ws/battle/:battleId
Local:      ws://localhost:8000/v1/battles/ws/battle/:battleId
```

**Messages received:**
```json
{ "type": "STATE_UPDATE", "battleId": "uuid", "round": 3, "state": { ... } }
{ "type": "BATTLE_END", "winnerId": "uuid", "eloChange": {...} }
```

---

```http

}
```

## Error Responses

All errors follow this shape:

```json
{ "error": "Human readable message", "statusCode": 401 }
```

| Code | Meaning |
|------|---------|
| 400 | Invalid request body or missing fields |
| 401 | Missing or expired JWT |
| 403 | Forbidden (not your agent, or production dev-login) |
| 404 | Resource not found |
| 409 | Conflict (duplicate tx hash, etc.) |
| 500 | Internal error (check service logs) |

## Complete User Flow (summary)

```
1. POST /v1/auth/privy          → get accessToken + refreshToken
2. POST /v1/agents              → create agent + INFT mint (async)
3. GET  /v1/agents/:id/evolution → check evolution stage
4. POST /v1/token/bridge/deposit → record EVM deposit
5. GET  /v1/token/bridge/deposit/:id → poll until CONFIRMED
6. POST /v1/wallets/deposits    → credit agent wallet
7. GET  /v1/token/price         → check $ARENA price
8. POST /v1/matchmaking         → join ranked queue
9. GET  /v1/matchmaking/status/:agentId → wait for matchId
10. WS  /v1/battles/ws/battle/:id → connect for live updates
11. GET  /v1/battles/:id        → get result when COMPLETED
12. POST /v1/agents/:id/train   → queue training with battle replay
13. GET  /v1/leaderboard/global → check your rank
```
