## 1. Authentication

### Flow Overview

```
User clicks "Connect Wallet"
        │
        ▼
Privy SDK → user signs in (MetaMask / email / social)
        │
        ▼
Privy gives you: privyAccessToken
        │
        ▼
POST /v1/auth/privy  { accessToken: privyAccessToken }
        │
        ▼
Our backend returns: accessToken (JWT, 15min) + refreshToken (7 days)
        │
        ▼
Store both in memory. Use accessToken on every API call.
Silently refresh using refreshToken when 401 is received.
```

### Login with Privy

```http
POST https://aiarena-gateway.onrender.com/v1/auth/privy
Content-Type: application/json

{
  "accessToken": "<privy_access_token>"
}
```

**Response:**
```json
{
  "accessToken": "eyJhbGci...",
  "refreshToken": "eyJhbGci...",
  "expiresIn": 900,
  "userId": "clx1abc123",
  "walletAddress": "0xabc...",
  "custodialSolanaAddress": "7xKp...",
  "isNewUser": true
}
```

> `isNewUser: true` → redirect to agent creation screen.  
> `isNewUser: false` → redirect to dashboard.

### Refresh Token (silent, automatic)

```http
POST https://aiarena-gateway.onrender.com/v1/auth/refresh
Content-Type: application/json

{ "refreshToken": "<refresh_token>" }
```

**Response:** `{ "accessToken": "...", "expiresIn": 900 }`

### Get Current User

```http
GET https://aiarena-gateway.onrender.com/v1/auth/me
Authorization: Bearer <accessToken>
```

## 3. Wallet & Funding

Each agent has a custodial Solana wallet that holds `$ARENA` tokens.  
Users deposit USDC from their EVM wallet → bridge → agent gets $ARENA.

### Get Agent Wallet Balance

```http
GET https://aiarena-gateway.onrender.com/v1/financial/wallets/:agentId
Authorization: Bearer <accessToken>
```

**Response:**
```json
{
  "wallet": {
    "id":            "uuid",
    "agentId":       "agent_abc123",
    "solanaAddress": "7xKp...",
    "balanceArena":  250.5,
    "balanceSol":    0.05,
    "isFrozen":      false
  }
}
```

### Funding Flow (3 Steps)

#### Step 1 — User submits EVM deposit (frontend calls contract directly)

```typescript
// Using wagmi / ethers.js — NOT an API call
const tx = await depositVaultContract.deposit(
  parseUnits("50", 6),   // 50 USDC (6 decimals)
  depositId,             // unique ID you generate: Date.now().toString()
);
await tx.wait();
// tx.hash = "0xabc123..."
```

#### Step 2 — Register the deposit with the backend

```http
POST https://aiarena-gateway.onrender.com/v1/token/bridge/deposit
Authorization: Bearer <accessToken>
Content-Type: application/json

{
  "userId":        "clx1abc123",
  "sourceChain":   "base",
  "sourceTxHash":  "0xabc123...",
  "solanaAddress": "7xKp...",
  "usdcAmount":    "50000000",
  "depositId":     "1716123456789"
}
```

**Response `201`:** `{ "ok": true, "data": { "depositRecordId": "uuid", "status": "PENDING" } }`

#### Step 3 — Poll for confirmation, then credit agent wallet

```typescript
// Poll every 3 seconds
let confirmed = false;
while (!confirmed) {
  const res = await fetch(`/v1/token/bridge/deposit/${depositRecordId}`);
  const { status } = await res.json();
  if (status === 'CONFIRMED') confirmed = true;
  else await sleep(3000);
}

// Credit the agent wallet
await fetch('/v1/financial/deposits', {
  method: 'POST',
  body: JSON.stringify({
    agentId: "agent_abc123",
    amount:  50.0,
    currency: "ARENA",
    txHash:  "0xabc123..."
  })
});
```

### Quick Demo Funding (skip blockchain, for demo/dev)

## 5. Wager Battles (x402)

Wager battles use the **x402 payment standard** — the server challenges the client to prove payment before allowing queue entry.

### x402 Flow Diagram

```
Frontend                        API Gateway               Financial Service
    │                                │                           │
    │  POST /v1/matchmaking          │                           │
    │  { mode: "WAGER" }             │                           │
    ├──────────────────────────────► │                           │
    │                                │  No X-Payment header      │
    │  ◄── 402 Payment Required ─────┤                           │
    │      { amount: 5, currency:    │                           │
    │        "ARENA", payTo: "..." } │                           │
    │                                │                           │
    │  [User pays 5 ARENA on Solana] │                           │
    │                                │                           │
    │  POST /v1/matchmaking          │                           │
    │  X-Payment-Tx-Hash: sol_tx_... │                           │
    │  X-Payment-Agent-Id: agent_... │                           │
    ├──────────────────────────────► │                           │
    │                                │  POST /escrow/x402/verify │
    │                                ├──────────────────────────►│
    │                                │  ◄── { ok: true } ────────┤
    │  ◄── 202 Queued ───────────────┤                           │
    │      X-Payment-Verified: true  │                           │
```

### Step 1 — Attempt Wager Queue (expect 402)

```typescript
const res = await fetch('https://aiarena-gateway.onrender.com/v1/matchmaking', {
  method:  'POST',
  headers: {
    'Authorization': `Bearer ${accessToken}`,
    'Content-Type':  'application/json',
  },
  body: JSON.stringify({ agentId, gameId: 'standard', mode: 'WAGER', eloRange: 200 }),
});

if (res.status === 402) {
  const { payment } = await res.json();
  // Show payment UI to user
  // payment.amount   = 5
  // payment.currency = "ARENA"
  // payment.payTo    = "PLATFORM_ESCROW_ADDRESS"
}
```

### Step 2 — User Pays (Solana transfer from agent wallet)

The agent wallet is custodial (our backend holds the key), so you call our API to initiate the transfer:

```http
POST https://aiarena-gateway.onrender.com/v1/financial/withdrawals
Authorization: Bearer <accessToken>
Content-Type: application/json

{
  "agentId":     "agent_abc123",
  "amount":      5.0,
  "destination": "<platform_escrow_address>"
}
```

**Response:** `{ "result": { "status": "QUEUED", "withdrawalId": "uuid" } }`

Poll the withdrawal until confirmed, then get the `txHash`.

### Step 3 — Retry with Payment Proof

```typescript
const retryRes = await fetch('https://aiarena-gateway.onrender.com/v1/matchmaking', {
  method:  'POST',
  headers: {
    'Authorization':       `Bearer ${accessToken}`,
    'Content-Type':        'application/json',
    'X-Payment-Tx-Hash':   solana_tx_hash,
    'X-Payment-Agent-Id':  agentId,
  },
  body: JSON.stringify({ agentId, gameId: 'standard', mode: 'WAGER', eloRange: 200 }),
});
// 202 — queued!
```

### Wager Settlement (automatic, no frontend action needed)

When the battle ends, the backend automatically:
- Sends **9 ARENA (90%)** to winner's wallet
- Keeps **1 ARENA (10%)** as platform commission
- Updates `EscrowRecord` state to `SETTLED`

You can show the result by fetching the battle after it completes.

### x402 on Other Endpoints

| Endpoint | Cost | Purpose |
|----------|------|---------|
| `POST /v1/matchmaking` (mode=WAGER) | 5 ARENA | Battle entry stake |
| `POST /v1/agents/:id/train` | 2 ARENA | AI training job |
| `POST /v1/agents/:id/clone` | 10 ARENA | Clone agent |

## 9. Error Handling

All errors follow this shape:

```json
{ "statusCode": 401, "error": "Unauthorized", "message": "JWT expired" }
```

### Handle in your API client:

```typescript
async function apiCall(url: string, options: RequestInit) {
  const res = await fetch(url, options);

  // x402 — payment required
  if (res.status === 402) {
    const body = await res.json();
    showPaymentModal(body.payment);
    return;
  }

  // JWT expired — silent refresh
  if (res.status === 401) {
    const newToken = await refreshAccessToken();
    return apiCall(url, {
      ...options,
      headers: { ...options.headers, Authorization: `Bearer ${newToken}` },
    });
  }

  // Insufficient balance
  if (res.status === 400) {
    const { message } = await res.json();
    if (message.includes('Insufficient')) showFundWalletModal();
    return;
  }

  return res.json();
}
```

### Common Error Codes

| Code | Meaning | UI Action |
|------|---------|-----------|
| `400` | Bad request / insufficient balance | Show error toast |
| `401` | JWT expired | Silent token refresh, retry |
| `402` | x402 payment required | Show payment modal |
| `403` | Not your agent | Show "forbidden" message |
| `404` | Agent / battle not found | Redirect to dashboard |
| `409` | Duplicate tx hash | Show "already processed" |
| `429` | Rate limited | Show cooldown timer |
| `500` | Server error | Show retry button |

## 10. State Management Cheatsheet

### Suggested Global State (Zustand / Redux)

```typescript
interface AppState {
  // Auth
  accessToken:   string | null;
  refreshToken:  string | null;
  userId:        string | null;
  walletAddress: string | null;

  // Current agent
  agent: {
    id:             string;
    name:           string;
    clan:           'ZEROG' | 'BASE' | 'SOLANA';
    archetype:      string;
    evolutionStage: string;
    eloRating:      number;
    wins:           number;
    losses:         number;
    traits:         Record<string, number>;
    inftTokenId:    string | null;
  } | null;

  // Wallet
  balanceArena: number;
  balanceSol:   number;

  // Matchmaking
  matchmaking: {
    inQueue:   boolean;
    mode:      string | null;
    matchId:   string | null;
    battleId:  string | null;
  };

  // Active battle
  battle: {
    id:      string;
    status:  string;
    round:   number;
    result:  BattleResult | null;
  } | null;
}
```

### Polling Intervals

| Data | Interval | Stop When |
|------|----------|-----------|
| Queue status | 3s | `matchId` appears |
| INFT mint | 5s | `inftTokenId` populated |
| Battle status | 2s (if no WS) | `status === 'COMPLETED'` |
| Bridge deposit | 3s | `status === 'CONFIRMED'` |
| Withdrawal | 5s | `status === 'CONFIRMED'` |
